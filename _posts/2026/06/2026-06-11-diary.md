```raw

declare -r output="${@: -1}"
                  ^--------^ SC2124 (warning): Assigning an array to a string! Assign as array, or use * instead of @ to concatenate.
```

SC2124 の検出は適切でない気もする。配列の最後の要素を文字列として変数に代入した
いのは明白ではないか。

```raw
In pdf-optimize line 18:
gs $gs_options -sOutputFile="$workpath" "$input"
   ^---------^ SC2086 (info): Double quote to prevent globbing and word splitting.

Did you mean:
gs "$gs_options" -sOutputFile="$workpath" "$input"
```

変数の値はこうだった：

```bash
declare -r gs_options="-sDEVICE=pdfwrite -dCompatibilityLevel=1.4 -dPDFSETTINGS=/printer -dNOPAUSE -dBATCH"
```

変数を配列に書き換えて `${gs_options[@]}` を引数とする。

次のは想像がつかない：

```raw
In video-overlap-durations line 16:
    durations+=($(ffprobe "${ffprobe_options[@]}" "$i"))
                ^-- SC2207 (warning): Prefer mapfile or read -a to split command output (or quote to avoid splitting).
```

配列 `durations` の末尾に `ffprobe` の出力で拡張していく。[ShellCheck] を黙らせ
るには一時変数を設けざるを得ない。
